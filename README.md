# Game Vault

Game Vault is a web application for managing personal game libraries, writing reviews, and tracking user activity. The system is built as a set of Spring Boot microservices with a Vue.js frontend and asynchronous communication based on RabbitMQ.

## Architecture

The application consists of the following services:

| Service               | Responsibility                                             |
| --------------------- | ---------------------------------------------------------- |
| gateway-service       | Entry point for client requests and request routing        |
| user-service          | User management and authentication                         |
| game-service          | Game library management, reviews, and RAWG API integration |
| notification-service  | Notification handling                                      |
| user-insights-service | Activity tracking and analytics                            |
| common-types          | Shared events and DTOs                                     |

### Design Decisions

The project was intentionally designed as a microservices-based system to explore common patterns used in distributed applications and to separate concerns between business domains.

Key architectural decisions include:

* **API Gateway** provides a single entry point for frontend communication and service routing.
* **RabbitMQ** enables asynchronous communication between services, reducing direct dependencies and improving scalability.
* **Dedicated databases per service** prevent tight coupling between domains and allow independent evolution of data models.
* **User Insights Service** is isolated from operational services to separate analytical workloads from transactional workloads.
* **Shared event contracts** in the `common-types` module ensure consistency in inter-service communication while keeping services independently deployable.

This architecture introduces additional operational complexity compared to a monolithic application, but it provides a practical environment for learning distributed systems concepts such as event-driven communication, service boundaries, eventual consistency, and independent deployment.

## Configuration

Create a `.env` file in the project root:

```env
RAWGIO_API_KEY=your_api_key
```

A RAWG API key can be obtained from:

https://rawg.io/apidocs

## Running the Application

Start the entire environment:

```bash
docker compose up --build
```

## Event Communication

Services communicate asynchronously through RabbitMQ. Domain events generated by application services are consumed by notification and analytics components.

Examples include:

* user registration events
* game collection updates
* review creation and deletion
* user activity tracking
